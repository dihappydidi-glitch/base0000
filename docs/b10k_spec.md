# Десятитысячно-единичная система счисления (base-10000)
## Спецификация

---

## 1. Назначение

Десятитысячно-единичная система счисления (base-10000 / B10K) предназначена
для **представления и вычислений над сколь угодно большими десятичными числами**
без конвертации в машинный целочисленный тип (`int`).

Ключевое свойство: **разрядность не ограничена**. Число может содержать
миллионы, миллиарды и более разрядов — лишь бы хватило памяти.
Все арифметические операции выполняются поразрядно, над массивами цифр 0…9999.

---

## 2. Внешний формат строки

```
[знак]левая_половина : правая_половина
```

| Элемент | Обязателен? | Описание |
|---------|-------------|----------|
| `знак` | нет | `-` для отрицательных чисел |
| `левая_половина` | да¹ | одна или более 4-цифровых групп, разделённых `.` |
| `:` | да | ровно одно двоеточие |
| `правая_половина` | да | одна или более 4-цифровых групп, разделённых `.` |

¹ Если половина не содержит ни одной группы, она считается равной `0`.

**Правила:**
- Каждая группа — ровно 4 десятичные цифры (с ведущими нулями), значение 0…9999.
- Левая и правая половины имеют **одинаковое число групп**.
- Группы внутри половины разделяются точками `.`.
- Ровно одно двоеточие `:` между половинами.

### 2.1. Математическая модель — переплетение (interleaving)

Цифры в big-endian представлении переплетаются между половинами:

```
BE = [L₀, R₀, L₁, R₁, …, Lₙ₋₁, Rₙ₋₁]

       2n−1           2n−2              0
total = L₀·B    + R₀·B    + … + Rₙ₋₁·B

где B = 10000, n = число пар, L₊ — левые группы, R₊ — правые группы
```

Или эквивалентно: **чётные индексы BE → левая половина, нечётные → правая**.

### 2.2. Примеры строк

| Строка | BE-цифры | Всего |
|--------|----------|-------|
| `0000:0005` | [0, 5] | 5 |
| `0001:0000` | [1, 0] | 10 000 |
| `9999:9999` | [9999, 9999] | 99 999 999 |
| `0000.0001:0000.0000` | [0, 0, 1, 0] | 10 000 |
| `0000.0000:0001.0000` | [0, 1, 0, 0] | 100 000 000 |
| `9999.9999:0000.0000` | [9999, 0, 9999, 0] | 9 999 000 099 990 000 |
| `-0000:0005` | [0, 5] | −5 |

### 2.3. Каноническая форма

При форматировании система приводит число к **канонической форме**:

1. Little-endian разряды → big-endian
2. Дополнить до чётного числа групп ведущим `0` при необходимости
3. Разбить на пары: (BE[0], BE[1]), (BE[2], BE[3]), …
4. Первый элемент каждой пары → левая половина, второй → правая
5. Каждая группа → `%04d`, соединить через `.` внутри половины, половины через `:`

## 3. Внутреннее представление

Число хранится как структура:

```python
@dataclass
class B10K:
    sign: int        # 1 или -1
    digs: List[int]  # little-endian массив разрядов 0..9999
```

### 3.1. Little-endian разряды

Массив `digs` — это цифры числа в системе с основанием 10 000,
записанные от младшего разряда к старшему:

```
digs[0] + digs[1]·B¹ + digs[2]·B² + … + digs[n-1]·Bⁿ⁻¹
```

где B = 10000.

### 3.2. Сокращение (trim)

Массив не содержит ведущих нулей (кроме самого нуля: `[0]`).
Это значит, что `digs[-1] ≠ 0` для любого ненулевого числа.

### 3.3. Знак

- `sign = 1` для положительных чисел и нуля
- `sign = -1` для отрицательных чисел
- Ноль всегда `sign = 1`

## 5. Поразрядные операции (алгоритмы)

### 5.1. Сложение `_add_abs(a, b)`

```
res = []
carry = 0
для i от 0 до max(len(a), len(b))-1:
    сумма = a[i] + b[i] + carry    (0, если i за границей)
    res[i] = сумма % 10000
    carry = сумма // 10000
если carry: res.append(carry)
вернуть _trim(res)
```

Сложность: O(N), где N = max(|a|, |b|).

### 5.2. Вычитание `_sub_abs(a, b)` (a ≥ b)

```
res = []
borrow = 0
для i от 0 до len(a)-1:
    разность = a[i] - (b[i] или 0) - borrow
    если разность < 0:
        разность += 10000; borrow = 1
    иначе: borrow = 0
    res[i] = разность
вернуть _trim(res)
```

Сложность: O(N).

### 5.3. Умножение `_mul_abs(a, b)` — столбиком

```
res = [0] × (len(a) + len(b))
для i, da в a:
    carry = 0
    для j, db в b:
        произведение = res[i+j] + da·db + carry
        res[i+j] = произведение % 10000
        carry = произведение // 10000
    если carry: res[i+len(b)] += carry
вернуть _trim(res)
```

Сложность: O(N·M) = O(N²) для чисел равной длины.

### 5.4. Деление `_div_mod_abs(a, b)` — уголком

```
a_be = reversed(a)    # big-endian для обхода
b_top = b[-1]         # старший разряд делителя

quot = []             # частное (big-endian)
rem_be = []           # остаток (big-endian)

для da в a_be:
    rem_be.append(da)
    если |rem_be| < |b|:
        quot.append(0); continue
    
    # прикидка цифры частного
    r_hi = rem_be[0] × 10000 + rem_be[1] (если есть)
    guess = r_hi // b_top (не больше 9999)
    
    # коррекция: подбор максимального guess,
    # при котором guess·b ≤ rem
    пока guess·b > rem: guess -= 1
    
    rem = rem - guess·b
    quot.append(guess)

вернуть (reversed(quot), rem)
```

Сложность: O(N·M), в среднем O(N).

### 5.5. Сдвиги

| Операция | Действие | LE-реализация |
|----------|----------|---------------|
| `shift_left(a, k)` | a × 10000ᵏ | `[0]×k + a.digs` |
| `shift_right(a, k)` | a // 10000ᵏ | `a.digs[k:]` |

Сдвиг влево — дописывает нули МЛАДШИМИ разрядами (умножение на степень основания).
Сдвиг вправо — отбрасывает k младших разрядов (целочисленное деление на степень основания).

## 6. Основные операции

### 6.1. neg(a) — смена знака

Если `a = 0`, возвращает `0`. Иначе возвращает число с тем же модулем и обратным знаком.

### 6.2. add(a, b)

```
если знаки совпадают:
    результат = (знак, _add_abs(a, b))
иначе:
    сравнить модули
    если |a| = |b|: вернуть 0
    если |a| > |b|: (знак a, _sub_abs(a, b))
    иначе:          (знак b, _sub_abs(b, a))
```

### 6.3. sub(a, b)

`sub(a, b) = add(a, neg(b))`

### 6.4. mul(a, b)

```
если a = 0 или b = 0: вернуть 0
знак: + если знаки совпадают, − если разные
результат: (знак, _mul_abs(a, b))
```

### 6.5. div_mod(a, b) — Евклидово деление

Гарантирует:
```
a = q·b + r, где 0 ≤ r < |b|
```

Алгоритм:
1. Вычислить частное и остаток через `_div_mod_abs(|a|, |b|)`
2. Частное = `sign(a)·sign(b)`, остаток = `sign(a)`
3. **Евклидова коррекция**: если остаток отрицательный → вычесть 1 из частного,
   прибавить `|b|` к остатку

### 6.6. pow_b10k(a, b) — возведение в степень

Бинарное возведение (exponentiation by squaring) для целого неотрицательного
показателя:

```
result = 1
base = a
exp = b
пока exp > 0:
    если exp нечётный: result ×= base
    base ×= base
    exp //= 2
вернуть result
```

Сложность: O(log₂(b)) умножений.

## 7. Дополнительные операции

### 7.1. fact(n) — факториал

```
n! = 1 × 2 × 3 × … × n
```

### 7.2. gcd(a, b) — наибольший общий делитель

Алгоритм Евклида:
```
пока b ≠ 0: a, b = b, a mod b
вернуть a
```

### 7.3. lcm(a, b) — наименьшее общее кратное

```
lcm(a, b) = |a·b| / gcd(a, b)
```

### 7.4. isqrt(a) — целочисленный квадратный корень

Метод Ньютона (вавилонский метод):

1. Начальное приближение: подбираем `x` так, что `x² > a`
2. `x = (x + a/x) / 2` до сходимости
3. Коррекция: уменьшаем, пока `x² > a`; увеличиваем, пока `(x+1)² ≤ a`

## 8. Парсинг и форматирование

### 8.1. parse(s) — строка → B10K

1. Удалить BOM и нецифровые символы в начале
2. Извлечь знак (`−` в начале)
3. Разделить по `:` на левую и правую половины
4. Разбить каждую половину по `.` на 4-цифровые группы
5. Валидировать каждую группу: 0…9999
6. Переплести: `BE = [L₀, R₀, L₁, R₁, …]`
7. Перевернуть в LE: `digs = reversed(BE)`, обрезать ведущие нули
8. Вернуть `B10K(sign, digs)`

### 8.2. format_num(a) — B10K → строка

1. Если 0 → `"0000:0000"`
2. LE-разряды → BE: `be = reversed(a.digs)`
3. Дополнить до чётного числа групп: `be.insert(0, 0)` если нечётное
4. Разбить на пары: левые = чётные индексы BE, правые = нечётные
5. Каждую группу → `f"{g:04d}"`, соединить через `.` внутри половины
6. Соединить половины через `:`

## 10. Сравнение чисел

Сравнение сначала по знаку, затем по модулю.
Сравнение модулей — сравнение длины LE-массива, затем поразрядно от старшего
к младшему.

| Операция | Реализация |
|----------|------------|
| `a == b` | знаки равны И модули равны |
| `a < b`  | если знаки разные: минус < плюс |
|          | если знаки равны: |a| < |b| × sign |
| `a > b`  | `not (a <= b)` |

## 11. Особые случаи

| Ситуация | Поведение |
|----------|-----------|
| Деление на ноль | `ZeroDivisionError` |
| Отрицательная степень | `ValueError` |
| Квадратный корень из отрицательного | `ValueError` |
| Факториал отрицательного | `ValueError` |
| Пустая строка при парсинге | `ValueError` |
| Группа вне 0…9999 | `ValueError` |
| Ноль (0) | `B10K(sign=1, digs=[0])` → `"0000:0000"` |
| Сдвиг нуля | возвращает 0 |
| Результат 0 после операции | знак всегда +1 |

## 13. Ограничения

- Работает только с целыми числами (не дробными)
- Степень поддерживается только для неотрицательного показателя
- Производительность: O(N²) для умножения и деления длинных чисел
- Все значения хранятся в памяти как Python-списки int'ов
