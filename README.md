# base0000 — Десятитысячно-единичная система счисления (B10K)

Система для представления и вычислений над **сколь угодно большими десятичными числами** без конвертации в машинный `int`. Все операции выполняются поразрядно над массивами цифр 0…9999, что позволяет работать с числами любой длины — миллионы и миллиарды разрядов, ограничение только по памяти.

Поддерживаются как **целые**, так и **дробные** числа с произвольной точностью.

## Формат чисел

### Целые числа

```
[знак]левая_половина : правая_половина
```

| Элемент | Описание |
|---------|----------|
| `знак` | `-` для отрицательных чисел |
| `левая_половина` | одна или более 4-цифровых групп (0…9999), разделённых `.` |
| `:` | ровно одно двоеточие |
| `правая_половина` | одна или более 4-цифровых групп (0…9999), разделённых `.` |

Правила:
- Каждая группа — ровно 4 десятичные цифры с ведущими нулями
- Левая и правая половины имеют **одинаковое число групп**
- Минимальная запись числа — одна пара `0000:0000` (ноль)

### Дробные числа

Формат — то же самое `левая_половина : правая_половина`, но с **запятой** как десятичным разделителем. Дробные разряды группируются **с правого конца (с младших)** по 8 цифр — каждая 8-ка даёт пару (L, R):

```
# целая часть в левой половине
[знак]целая_часть , левая_дробь : правая_дробь

# целая часть в правой половине
[знак]левая_дробь : целая_часть , правая_дробь
```

| Элемент | Описание |
|---------|----------|
| `знак` | `-` для отрицательных чисел |
| `целая_часть` | цифры, непосредственно примыкающие к запятой |
| `,` | десятичный разделитель (европейская нотация) |
| `левая_дробь / правая_дробь` | группы цифр, разделённые `.` |
| `:` | ровно одно двоеточие |

Правила:
- Запятая `,` — десятичный разделитель; целая часть — группа цифр, примыкающая к запятой
- Все остальные группы (в левой и правой половинах) — **дробные разряды**
- Первая группа после запятой может быть короче 4 цифр (ведущие нули опущены)
- Остальные группы — ровно по 4 цифры

### Принцип переплетения (interleaving)

**Big-endian разряды (группы 0…9999) переплетаются между половинами:**

```
BE = [L₀, R₀, L₁, R₁, L₂, R₂, …, Lₙ₋₁, Rₙ₋₁]
```

где L₊ — группы левой половины, R₊ — группы правой половины, каждая — значение 0…9999.

| Индексы BE | Половина |
|------------|----------|
| 0, 2, 4, … (чётные) | левая половина |
| 1, 3, 5, … (нечётные) | правая половина |

**Для целых чисел** значение вычисляется как сумма по степеням 10000:

```
число = L₀·B²ⁿ⁻¹ + R₀·B²ⁿ⁻² + … + Lₙ₋₁·B¹ + Rₙ₋₁·B⁰
```

где B = 10000, n — число пар.

**Для дробных чисел** используется **конкатенация десятичных цифр**:

1. Каждая группа BE[i] дополняется слева до 4 цифр: `pad4(BE[i])`
2. Все 4-цифровые строки сцепляются в одну
3. Ведущие нули удаляются — полученная строка и есть десятичные цифры дробной части

Математически:
```
s = concat(pad4(BE[0]), pad4(BE[1]), …, pad4(BE[n-1]))
s' = s.lstrip('0')       # удалить ведущие нули
дробная_часть = 0.s'     # интерпретация как десятичной дроби
```

B10K хранит дробную часть **не как сумму по степеням 10000**, а как **десятичные цифры**, разбитые на группы по 4 и переплетённые между левой и правой половинами.

**Алгоритм форматирования (число → B10K):**

```
LE-разряды → развернуть в BE
→ если нечётное число BE-групп, добавить 0000 в начало
→ BE[чётные] → левая половина
→ BE[нечётные] → правая половина
→ склеить: левая_половина : правая_половина
```

### Направление обработки: «с конца, с младших разрядов»

**Ключевой принцип:** все преобразования между десятичной записью и B10K выполняются, начиная с **младших (самых правых) разрядов**.

**Алгоритм для дробных (десятичные цифры → B10K):**

```
дробные_цифры (строка десятичных цифр после запятой)

1. Обрабатываем СПРАВА НАЛЕВО шагами по 8 цифр:
   — каждая 8-ка даёт одну пару (Lᵢ, Rᵢ)
   — первые 4 цифры 8-ки → левая группа (Lᵢ)
   — последние 4 цифры 8-ки → правая группа (Rᵢ)

2. Самая левая (старшая) группа может быть короче 8 цифр
   → дополняем её слева нулями до 8

3. Первая левая группа — без ведущих нулей

4. Склеить:  целая, L₀.L₁…Lₙ₋₁ : R₀.R₁…Rₙ₋₁
```

Или эквивалентно (обратная совместимость):
1. Дополнить дробные цифры слева нулями до длины, кратной 4
2. Разбить на 4-цифровые группы (слева направо)
3. Чётные группы → левая половина, нечётные → правая
4. Первая левая группа — без ведущих нулей

## Преимущества B10K

### 1. Произвольная точность без ограничений машинного int
B10K работает с массивами цифр 0…9999 — длина числа ограничена только доступной памятью. Никаких переполнений (`overflow`), никаких `maxint`. Числа в миллионы и миллиарды разрядов — штатный режим, не особый случай.

### 2. Нативно-десятичное представление
В отличие от двоичного big integer (GMP, Python `int`), B10K хранит числа **в десятичной системе счисления**. Это значит:
- **Нет ошибок конвертации** двоичная ↔ десятичная — перевод 2→10 для big int есть O(n²), и с ростом числа он становится узким горлышком.
- **Нет ошибок округления десятичных дробей** — 0,1 представляется *точно*, в отличие от двоичной floating-point арифметики (IEEE 754), где 0,1 — периодическая дробь.
- Входные данные из предметных областей (финансы, бухгалтерия, налоги) и так десятичны — B10K работает с ними напрямую.

### 3. Человеко-читаемые группы по 4 цифры
Формат `0001.6789:2345.0123` нагляден: каждая 4-цифровая группа легко воспринимается глазом. Не нужно мысленно разбивать длинное число `1234567890123` на разряды — B10K уже разбит.

### 4. Переплетающийся формат (interleaving)
Две половины числа, переплетённые разряд за разрядом, дают:
- **Визуальную симметрию** — левая и правая половины зеркально отражают друг друга.
- **Компактность** — число из 8 групп (32 десятичных цифры) записывается всего в 2 строках по 4 группы.
- **Наглядный контроль переноса** — при сложении/умножении столбиком перенос идёт между соседними BE-группами, что легко отследить визуально.

### 5. Поразрядные операции с наглядным контролем
Все арифметические операции — над модулем числа разряд за разрядом, как вручную столбиком:
- Каждый разряд 0…9999 помещается в одну переменную (никаких «разбить 64-битное слово пополам»).
- Переносы и заёмы явны — их можно вывести на экран, отладить, проверить.
- Идеально для **обучения алгоритмам произвольной точности**: сложение столбиком, умножение, деление уголком, извлечение корня — каждый шаг прозрачен.

### 6. Никаких ошибок округления десятичных дробей
В отличие от `float`/`double`:
| Число | float (IEEE 754) | B10K |
|-------|------------------|------|
| 0,1 | 0,10000000000000000555… | 0,1 **точно** |
| ⅓ | 0,33333333333333331483… | 0,3333… — с произвольной точностью, сколько разрядов задано |

B10K не округляет дробные цифры — они хранятся как есть, сколько бы их ни было.

### 7. Подходит для финансовых и бухгалтерских расчётов
- Все операции — над целыми числами дробных разрядов (нет проблем «копеек в рубле»).
- Десятичная арифметика без сюрпризов: `0,10 + 0,20 = 0,30`, а не `0,30000000000000004`.
- Законодательство многих стран требует десятичного представления для налоговой отчётности — B10K изоморфен предметной области.

### 8. Образовательная ценность
B10K — это «живой учебник» арифметики произвольной точности:
- Открытый исходный код на Python (~800 строк).
- Нет магии: сложение — это школьное сложение столбиком.
- Можно пошагово трассировать любой алгоритм с выводом промежуточных состояний.
- Наглядно показывает разницу big-endian / little-endian, переплетение, конкатенацию дробных разрядов.

### 9. Аппаратная реализуемость
Формат B10K ориентирован на поразрядную обработку — каждая группа 0…9999 помещается в:
- 14 бит (если кодировать двоично-десятично, BCD);
- 16 бит (полуслово — для эффективных аппаратных умножителей);
- 1 «цифру» DEC64 или десятичного FP-устройства.

Это делает B10K кандидатом для аппаратной реализации десятичной арифметики произвольной точности (в отличие от binary big integer, где каждый разряд — полное машинное слово, и конвертация тоже аппаратно затратна).

### 10. Гигантские числа: результат, который иначе не узнать
B10K легко вычисляет факториалы, степени и константы, которые невозможно получить ни на калькуляторе, ни в обычных языках программирования. Например:

```
9999! = 9999 × 9998 × 9997 × … × 1
```

— число длиной **35 656 десятичных цифр**. Ни один обычный калькулятор его не покажет. В double/float — бесконечность. B10K вычисляет его за секунды и хранит поразрядно:

…
Конец:  …0000.0000.0000.0000.0000.0000.0000.0000.0000.0000.0000.0000.0000.0000.0000.0000
```

</details>

### 11. Лёгкая проверяемость
Каждый результат можно проверить **поразрядно, цифра за цифрой**, без написания дополнительного кода:
- Формат удобен для code review — глаз видит переносы и суммы.
- Тестирование: каждый арифметический тест проверяет разряды, а не `assertEqual(hex(...))`.
- Для дробных: конкатенация BE-групп легко превращается обратно в десятичные цифры простым `strip('0')`.

## Примеры

### Целые

| Формат | BE-цифры (4 цифры = 1 группа × 10000ⁱ) | Десятичное |
|--------|----------------------------------------|------------|
| `0000:0005` | [0, 5] | 5 |
| `0001:0000` | [1, 0] | 10 000 |
| `9999:9999` | [9999, 9999] | 99 999 999 |
| `0000.0001:0000.0000` | [0, 0, 1, 0] | 10 000 |
| `0000.0000:0001.0000` | [0, 1, 0, 0] | 100 000 000 |
| `0001.0000:0000.0000` | [1, 0, 0, 0] | 1 000 000 000 000 |
| `9999.9999:0000.0000` | [9999, 0, 9999, 0] | 9 999 000 099 990 000 |
| `-0000:0005` | — | −5 |

Разбор `0000.0001:0000.0000`:
```
левая = [0, 1]     правая = [0, 0]
BE = [0, 0, 1, 0]  →  0·10000³ + 0·10000² + 1·10000 + 0 = 10000
```

### Дробные

| Формат | Десятичное | Запятая в |
|--------|------------|-----------|
| `6,708.3249.0892.1006:2039.9369.2752.1938` | 6,7082039324993690892275210061938… (√45) | левой половине |
| `3,556.0449.8060.4622:8933.0062.0615.2389` | 3,5568933044900628060061546222389… (∛45) | левой половине |

**Подробный разбор √45 (запятая в левой половине):**

```
  целая_часть      левая половина               правая половина
 ┌──────────┐  ┌──────────────────────────┐  ┌──────────────────────────┐
 6          ,  708 . 3249 . 0892 . 1006  :  2039 . 9369 . 2752 . 1938
```

1. **Целая часть** — `6`.
2. **Левая половина дроби** — группы `708`, `3249`, `0892`, `1006` (значения [708, 3249, 892, 1006]).
3. **Правая половина дроби** — группы `2039`, `9369`, `2752`, `1938` (значения [2039, 9369, 2752, 1938]).

Дробные разряды — групповое переплетение:

```
  BE = [L₀, R₀, L₁, R₁, L₂, R₂, L₃, R₃]
     = [708, 2039, 3249, 9369, 892, 2752, 1006, 1938]

  Каждая группа → 4 цифры (pad4):
  "0708", "2039", "3249", "9369", "0892", "2752", "1006", "1938"

  Сцепить: "07082039324993690892275210061938"
  Убрать ведущий ноль: "7082039324993690892275210061938"

  Итого: 6,7082039324993690892275210061938… = √45
```

**Как получен этот B10K из десятичных цифр √45:**

```
  Дробные цифры √45 (после запятой): 7082039324993690892275210061938… (31 цифра)

  Обрабатываем СПРАВА (с младших разрядов) по 8 цифр:
  ¹⁴ ···· ⁷⁷   ²³  ···· ³⁰   ¹⁵  ···· ²²   ⁰⁷  ···· ¹⁴
  1006 1938     0892 2752     3249 9369     0708 2039     ← 8-ка → (L,R)
  └──┬──┘ └──┬──┘  └──┬──┘ └──┬──┘  └──┬──┘ └──┬──┘  └─┬─┘ └──┬──┘
   L₃   R₃    L₂   R₂    L₁   R₁    L₀   R₀
  
  Самая левая 8-ка (0708 2039) была короче (7 цифр → дополнена до 8 слева)
  
  Первая левая группа без ведущего нуля: 708

  Результат: 6, L₀.L₁.L₂.L₃ : R₀.R₁.R₂.R₃
           = 6,708.3249.0892.1006 : 2039.9369.2752.1938
```

**Разбор ∛45 (запятая в левой половине):**

```
  целая_часть     левая половина               правая половина
 ┌──────────┐  ┌──────────────────────────┐  ┌──────────────────────────┐
 3           ,  556 . 0449 . 8060 . 4622  :  8933 . 0062 . 0615 . 2389
```

1. **Целая часть** — `3`.
2. **Левая половина дроби** — группы `556`, `0449`, `8060`, `4622` (значения [556, 449, 8060, 4622]).
3. **Правая половина дроби** — группы `8933`, `0062`, `0615`, `2389` (значения [8933, 62, 615, 2389]).

```
  BE = [556, 8933, 449, 62, 8060, 615, 4622, 2389]

  pad4: "0556", "8933", "0449", "0062", "8060", "0615", "4622", "2389"
  сцепить: "05568933044900628060061546222389"
  убрать ведущий ноль: "5568933044900628060061546222389"

  Итого: 3,5568933044900628060061546222389… = ∛45
```

**Как получен этот B10K из десятичных цифр ∛45:**

```
  Дробные цифры ∛45 (после запятой): 5568933044900628060061546222389… (31 цифра)

  Обрабатываем СПРАВА (с младших разрядов) по 8 цифр:
  ⁰⁷ ···· ¹⁴   ¹⁵  ···· ²²   ²³  ···· ³⁰   ⁰⁰  ···· ⁰⁶
  0556 8933     0449 0062     8060 0615     4622 2389     ← 8-ка → (L,R)
  └─┬──┘ └──┬──┘  └──┬──┘ └──┬──┘  └──┬──┘ └──┬──┘  └──┬──┘ └──┬──┘
   L₀   R₀    L₁   R₁    L₂   R₂    L₃   R₃
    
  Первая левая группа без ведущих нулей: 556

  Результат: 3, L₀.L₁.L₂.L₃ : R₀.R₁.R₂.R₃
           = 3,556.0449.8060.4622 : 8933.0062.0615.2389
```

### Арифметика

```
  0000:0005 + 0000:0003                        =  0000:0008
  0000:9999 + 0000:0001                        =  0001:0000
  9999:9999 + 0000:0001                        =  0000.0000:0001.0000
  0001:0000 - 0000:0001                        =  0000:9999
  0000:0005 - 0000:0003                        =  0000:0002
  
  0000:9999 × 0000:9999                        =  9998:0001
  9999:9999 × 9999:9999                        =  9999.0000:9998.0001
  0001:0000 × 0001:0000                        =  0000.0000:0001.0000
  9999.9999:0000.0000²                         =  9998.9996.9998.0000:0002.0002.0001.0000
  
  9998:0001 / 0000:9999                        =  0000:9999  (99 990 001 ÷ 9999 = 9999)
  0000:0010 / 0000:0003                        =  0000:0003  (10 ÷ 3 = 3)
  0000:0010 % 0000:0003                        =  0000:0001  (10 ÷ 3 = 3, ост. 1)

## Арифметические операции

| Оператор | Описание | Сложность |
|----------|----------|-----------|
| `a + b` | сложение | O(N) |
| `a - b` | вычитание | O(N) |
| `a * b` | умножение (столбиком) | O(N·M) |
| `a / b` | целочисленное деление | O(N·M) |
| `a // b` | целочисленное деление | O(N·M) |
| `a % b` | остаток (Евклидов: 0 ≤ r < \|b\|) | O(N·M) |
| `a ** b` | возведение в степень (b ≥ 0) | O(log₂(b)·M(NM)) |

Дополнительные функции:

| Функция | Описание |
|---------|----------|
| `fact(n)` | факториал n! |
| `gcd(a, b)` | наибольший общий делитель (алгоритм Евклида) |
| `lcm(a, b)` | наименьшее общее кратное |
| `isqrt(a)` | целочисленный квадратный корень (Ньютон) |
| `shift_left(a, k)` | сдвиг влево: a × 10000ᵏ |
| `shift_right(a, k)` | сдвиг вправо: a ∥ 10000ᵏ |

## Внутреннее устройство

### Представление числа

```python
@dataclass
class B10K:
    sign: int       # 1 или -1
    digs: List[int] # little-endian массив разрядов 0…9999
```

**Little-endian:** младший разряд по индексу 0.

```
значение = digs[0] + digs[1]·10000 + digs[2]·10000² + … + digs[n-1]·10000ⁿ⁻¹
```

**Trim:** массив не содержит ведущих нулей (кроме нуля: `[0]`).

**Знак:** `sign = 1` для положительных и нуля, `sign = -1` для отрицательных.
